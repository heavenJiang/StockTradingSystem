# 📈 AI 股票量化投资系统 (Stock Trading System)

本项目旨在开发一个基于 AI 机器学习模型的股票交易自动化验证与预测策略程序。系统的核心包含：真实历史数据获取、AI 智能特征工程与分类模型训练、严格止盈止损策略执行的历史回测，以及供人机交互的美观 Web UI。

---

## 🏗️ 模块架构与系统概览

### 1. 依赖配置与项目基础
项目依赖主要由 `requirements.txt` 统一管理，核心依赖包括：
- **akshare**: 获取A股历史全面且准确的前复权 K 线数据。
- **pandas, numpy**: 执行快速的数据处理、滚动窗口特征构造与技术指标运算。
- **scikit-learn**: 构建 AI 模型（采用了 `RandomForestClassifier` 随机森林模型）。该模型对表格型结构化数据的分类预测（涨/跌概率）具有极其优异的性能表现。
- **streamlit**: 构建现代化、响应式且美观大方的前端用户交互页面。
- **plotly**: 提供页面中的动态可交互数据可视化绘制。

### 2. [配置模块] `config.py`
处理系统运行参数的加载和保存。全局配置项落盘保存在本地的 `config.json` 文件中，包括：
- `rebuild_model`: 是否开启每次执行前重新训练 AI 模型。
- `target_stock_code`: 目标股票池代码（如 `"000001;600000"`）。
- `backtest_year`: 用于回溯测试和训练模型的历史年份数量。
- `stop_loss_threshold` / `stop_profit_threshold`: 严格的止损和止盈率（如 `0.10` 代表 10%）。
- `max_hold_days`: 最大持仓天数，超过天数后会触发强制平仓卖出。
- `init_capital`: 初始模拟回测资金池量。

### 3. [模型构建模块] `buildmodel.py`
- **数据获取**: 使用 `akshare.stock_zh_a_hist` 接口抓取日 K 线数据。
- **特征工程**: 通过日线滚动特征计算 `MACD`，各周期移动平均线 (MA5, MA10, MA20)，以及日均波动率等数据作为特征基底。
- **标签生成**: 基于前瞻性滑动窗口，推演未来不超过 `max_hold_days` 天内的真实高低点。如果在触碰止损前达到了止盈阈值（或者期满收益为正），则将当前节点标记为正样本（1 = 推荐买入）。反之则作为负样本（0 = 不买入）。
- **模型训练**: 将标准化后的数据喂给 `RandomForestClassifier` 模型，并在本地通过 `joblib` 将训练好的模型参数与数据缩放器（Scaler）打快照保存起来。

### 4. [回测验证模块] `backtest.py`
- 按时间步长遍历测试周期的 K 线数据。
- **交易逻辑**:
  - **买入**：当 AI 模型针对当日数据输出看涨信号且当前账户无持仓时，将剩余可用资金一并全仓吃进。
  - **卖出**：盘中探测。如果盘中最低价击穿配置的止损红线，则果断执行止损（哪怕遇到了极端跌停情况，系统也会保守按照当日最低价与当天收盘价去执行平仓）。如果盘中最高价触及止盈，则执行止盈；否则在持有达到 `max_hold_days` 时以收盘价原路出清。
- 生成格式化的高维交易日志字典并整合输出 DataFrame 对象供前端消费，输出内容包括总交易次数、整体胜率、历史投资回撤与回报率以及持仓周期等。

### 5. [链路统筹模块] `astock_AImodel.py`
负责将上述的所有散件能力封装成流水线。暴露 `run_pipeline()` 函数用以拉起整个自动化训练和回测链条；并且，在这个文件中还会获取最新的单个交易日数据推入训练完毕的模型当中，直接为下一个真实交易日提供明确的 **买卖操作建议**。

### 6. [可视化交互界面] `ui.py`
提供了一套基于 Streamlit 的无缝衔接式开箱即用 Web 界面。
- **参数控制区**: 在侧边栏供终端用户可视乎地修改持久化参数（例如变更持仓天数、改换个股代码），并自动落地入参至 JSON 文件。
- **信息看板面板**: 当回测被触发完毕后，系统不仅能够使用大数据卡片高亮胜率、投资回报率和资产等信息，还能展开交易全记录与渲染盈亏分布图等。

## 🛠️ 如何开发出的此项目 (开发纪实)

项目的顺利落成源于明确的需求转化和高效的 AI 工程辅助，在搭建的过程中分为以下几个核心阶段：

1. **需求提取与组件化拆分**：
   收到“利用 AI 模型构建股票回测程序，且须包含全链路买卖闭环”这项起因诉求后，我首先从需求文档中拆分和提炼出了 5 大可独立运作的工程化模块（`config`、`buildmodel`、`backtest`、`astock_AImodel`、`ui`）。并明确了通过经典的“随机森林分类器 (Random Forest)”作为智能判定中枢。
2. **底层数据设施与配置中心建设**：
   在不阻碍上下游进程的情况下优先敲定了包含所有控制变量的 `config.py`。随后成功搭建好通向真实市场数据的桥梁——引入了 `akshare`，它为模型训练输送了开箱即用的前复权A股 K 线骨干数据。
3. **特征加工、标签构造与模型训练难点攻坚**：
   在量化建模任务中，“如何客观地评价某一天是否值得买”实属业务核心逻辑的痛点。
   开发中刻意采取了保守且非常严苛的安全模拟策略：在系统底层的滑动窗口标注阶段，优先下探未来是否会触碰止损带（哪怕后面能拉升到天际，只要过程中被止损带击穿，就立刻将被评估的这天标记为“不该买的 0”，以防止极其有害的深水炸弹）。经历严格筛选的特征工程随后被流转进了 Sklearn 体系并沉淀成了离线的 `pkl` 模型文件。
4. **历史时间序列防未来数据泄露 (Data Leakage) 与回验策略**：
   搭建了高度拟真的 `backtest.py` 沙盘环境模块，按天为尺度线性回退推演，避免来自未来的信息提前被模型获取，保障策略的现实可行度。
5. **用户交互终端与可视化体验升华**：
   为了让严肃的系统不流于枯燥的控制台黑框打印，利用 `Streamlit` 取代了庞杂的前后端基建。我们在短时间内打造出了一组带有参数热更新、巨幅可视化看板（Plotly）并且对用户心智极度友好的交互台，最终让复杂的 AI 预测变成了页面上一句温暖的“明天到底该不该进场”指示呈现给用户。

## 👨‍💻 如何运行与验证

推荐在配置好 Python 环境并通过 `pip install -r requirements.txt` 安装完依赖后（如果您使用虚拟环境并且之前执行过安装，请确保使用的是该环境路径下的资源），直接在终端执行下述命令启动大盘：

```bash
streamlit run ui.py
```

在系统弹出的网页控制台中，设定好您想要分析回溯的“股票代码”以及期望评估的“回测年数”，随后狠狠点击**“🚀 开始模型训练与历史回测”**，系统就会自动完成从数据到推断的全套闭环并为您展示成果！
